# この記事の動機

CIのテストが長いと、レビューや修正のテンポが悪くなり、フラストレーションを感じることがあります。

かといって、ガードレールであるテストを減らすのは、慎重にならないといけないですし、時間がかかっていそうなテストを闇雲に修正しても時間が短縮されるとは限りません。

時間がかかっているテストケースが特定できれば、効率的にテスト時間の短縮のためのアクションを取ることができます。テストも、「推測するな、計測せよ」の考え方が生きます。

以前書いた記事では、PHPUnit10以降では廃止されてしまったEventHookを使っていたので、PHPUnit10以降で使える、Eventを使った形に書き換えて紹介します。

https://zenn.dev/lamp_inc/articles/ca1ae27dd6727c

# パッケージ

今回の内容をもとにパッケージをまとめたので、PHP8.1以上、PHPUnit10以上であれば、導入して使っていただけます。ぜひ試してみて、フィードバックをもらえると嬉しいです！

https://packagist.org/packages/takigawaakinori/phpunit-profiler

## 使い方

Composer でインストールします。

```bash
composer require --dev takigawaakinori/phpunit-profiler
```

`phpunit.xml`（または `phpunit.xml.dist`）の `<extensions>` に `TestProfilerExtension` を登録します。

```xml
<extensions>
    <bootstrap class="TakigawaAkinori\PhpunitProfiler\TestProfilerExtension"/>
</extensions>
```

この状態で `phpunit` を実行すると、デフォルトではテスト終了後に遅いテストの上位 20 件が標準出力に表示されます。

```text
Top 20 Slowest Tests:
--------------------------------------------------------------------------------
  1.  5.032s  Tests\Feature\ExampleTest::test_heavy_operation
  2.  3.210s  Tests\Feature\ExampleTest::test_api_call
  3.  0.523s  Tests\Unit\ExampleTest::test_calculation
--------------------------------------------------------------------------------
```

## 機能の紹介

`<parameter>` で挙動をカスタマイズできます。

```xml
<extensions>
    <bootstrap class="TakigawaAkinori\PhpunitProfiler\TestProfilerExtension">
        <parameter name="showPareto" value="true"/>
        <parameter name="slowThreshold" value="1.0"/>
        <parameter name="jsonOutput" value=".test-profile-result.json"/>
    </bootstrap>
</extensions>
```

| パラメータ | 型 | デフォルト | 説明 |
|-----------|----|-----------|------|
| `showTopN` | bool | `true` | 上位 N 件の遅いテスト一覧を表示。`false` で非表示 |
| `topCount` | int | `20` | 表示する遅いテスト件数 |
| `showPareto` | bool | `false` | パレート分析（上位 20% のテストが全体時間に占める割合）を表示 |
| `slowThreshold` | float | _(無効)_ | 指定秒数以上かかったテストを表示 |
| `jsonOutput` | string | _(無効)_ | 全テストの実行時間を JSON として出力するファイルパス |

JSON 出力を有効にすると、全テストの実行時間を機械可読な形式で保存できます。

# 実装方法

1. PHPUnitのExtensionを作成する
2. テスト開始イベントと終了イベントをサブスクライブし時間を記録する
3. テスト実行終了時に好きな形式で出力する

## PHPUnitのExtensionを作成する

PHPUnit 10 以降では、`PHPUnit\Runner\Extension\Extension` インターフェースを実装したクラスが Extension の入口になります。`bootstrap()` メソッドには `Configuration`・`Facade`・`ParameterCollection` が渡ってきます。

- `Configuration`: PHPUnit 全体の設定
- `Facade`: イベントサブスクライバを登録するための窓口
- `ParameterCollection`: `<parameter>` で与えられた値

```php
use PHPUnit\Runner\Extension\Extension;
use PHPUnit\Runner\Extension\Facade;
use PHPUnit\Runner\Extension\ParameterCollection;
use PHPUnit\TextUI\Configuration\Configuration;

final class TestProfilerExtension implements Extension
{
    public function bootstrap(
        Configuration $configuration,
        Facade $facade,
        ParameterCollection $parameters,
    ): void {
        // $parameters からオプションを取り出し、
        // $facade にサブスクライバを登録していく
    }
}
```

## テスト開始イベントと終了イベントをサブスクライブし時間を記録する

PHPUnit 10 以降のイベントは `PHPUnit\Event\*` 以下に定義されており、サブスクライブしたいイベントごとに対応する `*Subscriber` インターフェースが用意されています。テストの実行時間を計測するためには、以下の 2 つを使います。

- `PHPUnit\Event\Test\PreparedSubscriber`: テストの実行直前に発火する `Prepared` イベント
- `PHPUnit\Event\Test\FinishedSubscriber`: テスト終了時に発火する `Finished` イベント

`Prepared` と `Finished` の各イベントには `telemetryInfo()->time()` から `HRTime`（高精度時刻）が取れるので、開始時刻と終了時刻の差分を取れば実行時間になります。テストの同定には `event->test()->id()` を使います。

`HRTime` は `microtime()` とは異なり、システムクロックの変更の影響を受けないため、より正確な時間計測が可能です。パフォーマンス計測には `HRTime` を使うことが推奨されています。

https://www.php.net/manual/ja/function.microtime.php

```php
$facade->registerSubscribers(
    new class ($collector) implements PreparedSubscriber {
        public function __construct(private readonly TestTimeCollector $collector) {}

        public function notify(Prepared $event): void
        {
            $this->collector->recordStart(
                $event->test()->id(),
                $event->telemetryInfo()->time(),
            );
        }
    },
    new class ($collector) implements FinishedSubscriber {
        public function __construct(private readonly TestTimeCollector $collector) {}

        public function notify(Finished $event): void
        {
            $this->collector->recordEnd(
                $event->test()->id(),
                $event->telemetryInfo()->time(),
            );
        }
    },
);
```

注入している `TestTimeCollector` は、テストの開始と終了の時間を記録するクラスです。テスト ID をキーにして、経過時間を保存しています。

```php
final class TestTimeCollector
{
    /** @var array<string, HRTime> */
    private array $startTimes = [];

    /** @var array<string, float> */
    private array $durations = [];

    public function recordStart(string $testId, HRTime $time): void
    {
        $this->startTimes[$testId] = $time;
    }

    public function recordEnd(string $testId, HRTime $time): void
    {
        if (! isset($this->startTimes[$testId])) {
            return;
        }

        $duration = $time->duration($this->startTimes[$testId]);
        $this->durations[$testId] = $duration->asFloat();

        unset($this->startTimes[$testId]);
    }

    public function getResults(): TestDurationResultCollection
    {
        arsort($this->durations);

        $results = [];

        foreach ($this->durations as $testId => $seconds) {
            $results[] = new TestDurationResult($testId, $seconds);
        }

        return new TestDurationResultCollection(...$results);
    }
}
```

## テスト実行終了時に好きな形式で出力する

全テスト完了時に発火するのが `PHPUnit\Event\TestRunner\ExecutionFinished` イベントで、対応する `ExecutionFinishedSubscriber` をサブスクライブします。ここで集計結果を取り出し、好きな形式でフォーマットして出力します。

```php
new class ($collector, $outputter, $jsonWriter) implements ExecutionFinishedSubscriber {
    public function __construct(
        private readonly TestTimeCollector $collector,
        private readonly TestDurationOutputter $outputter,
        private readonly ?JsonResultWriter $jsonWriter,
    ) {}

    public function notify(ExecutionFinished $event): void
    {
        $results = $this->collector->getResults();
        $this->outputter->print($results);
        $this->jsonWriter?->write($results);
    }
},
```

例えば、以下のようなメソッドが `TestDurationOutputter` クラスにあって、設定した閾値より遅いテストの一覧を出力します。

```php
    public function print(TestDurationResultCollection $results): void
    {
        // ...(略)...

        // phpunit.xmlでパラメータが設定されていれば、$this->slowThreshold に値が入った状態でインスタンス化される
        if ($this->slowThreshold !== null) { 
            $this->printSlowThreshold($results);
        }
    }

    // 一例として閾値よりも遅いテストを出力する機能
    public function printSlowThreshold(TestDurationResultCollection $results): void
    {
        if ($results->isEmpty() || $this->slowThreshold === null) {
            return;
        }

        $slow = $results->slowerThan($this->slowThreshold);

        echo PHP_EOL . sprintf('Tests slower than %.3fs:', $this->slowThreshold) . PHP_EOL;
        echo str_repeat('-', 80) . PHP_EOL;

        if ($slow->isEmpty()) {
            echo '  (none)' . PHP_EOL;
        } else {
            $rank = 1;
            foreach ($slow as $result) {
                echo sprintf(
                    ' %2d. %6.3fs  %s',
                    $rank,
                    $result->durationInSeconds,
                    $result->testId,
                ) . PHP_EOL;
                $rank++;
            }
        }

        echo str_repeat('-', 80) . PHP_EOL;
    }
```
